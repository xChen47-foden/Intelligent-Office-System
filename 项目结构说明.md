# QXC 项目结构说明

## 项目概述
QXC 是一个前后端分离的智能办公应用，包含前端 Vue.js 应用和后端 API 服务。

## 目录结构

```
qxc/
├── frontend/                 # 前端项目
│   ├── src/                 # Vue.js 源代码
│   ├── public/              # 静态资源
│   ├── dist/                # 构建输出
│   ├── dist_electron/       # Electron 构建输出
│   ├── dist_installer/      # 安装包构建输出
│   ├── node_modules/        # Node.js 依赖
│   ├── package.json         # 前端依赖配置
│   ├── vite.config.ts       # Vite 配置
│   ├── tsconfig.json        # TypeScript 配置
│   ├── electron-main.js     # Electron 主进程
│   ├── preload.js           # Electron 预加载脚本
│   ├── build.js             # 构建脚本
│   └── 快速构建.bat         # 快速构建脚本
│
├── backend/                  # 后端项目
│   ├── fastapi_app/         # FastAPI 应用
│   ├── django/              # Django 应用
│   ├── node_service/        # Node.js 服务
│   ├── routes/              # 路由文件
│   ├── services/            # 服务层
│   ├── models/              # 数据模型
│   ├── migrations/          # 数据库迁移
│   ├── db/                  # 数据库文件
│   ├── uploads/             # 文件上传目录
│   ├── parsed/              # 解析文件
│   ├── server.js            # 主服务器文件
│   ├── server-production.js # 生产环境服务器
│   ├── docker-compose.yml   # Docker 配置
│   ├── Dockerfile           # Docker 镜像配置
│   ├── alembic.ini          # 数据库迁移配置
│   └── 部署相关脚本和文档
│
├── README.md                # 项目说明
├── README.zh-CN.md          # 中文项目说明
├── LICENSE                  # 许可证
├── .gitignore              # Git 忽略文件
├── .gitattributes          # Git 属性配置
└── compress-frontend-backend.ps1  # 压缩脚本
```

## 前端项目 (frontend/)

### 技术栈
- **框架**: Vue 3 + TypeScript
- **构建工具**: Vite
- **UI 框架**: 自定义组件库
- **桌面应用**: Electron
- **包管理**: npm

### 主要功能
- 用户界面和交互
- 数据可视化
- 文件上传和管理
- 实时通信
- 桌面应用支持

### 构建和运行
```bash
cd frontend
npm install
npm run dev          # 开发环境
npm run build        # 构建生产版本
npm run electron     # 运行桌面应用
```

## 后端项目 (backend/)

### 技术栈
- **API 框架**: FastAPI + Django
- **数据库**: SQLite
- **文件处理**: Python + Node.js
- **部署**: Docker + 云服务

### 主要功能
- RESTful API 服务
- 文件上传和处理
- 数据库管理
- 用户认证
- 业务逻辑处理

### 运行方式
```bash
cd backend
# Python 环境
python server.py

# Node.js 环境
node server.js

# Docker 环境
docker-compose up
```

## 压缩和部署

### 使用压缩脚本
```powershell
# 运行压缩脚本
.\compress-frontend-backend.ps1
```

### 手动压缩
```powershell
# 压缩前端
Compress-Archive -Path "frontend\*" -DestinationPath "frontend.zip"

# 压缩后端
Compress-Archive -Path "backend\*" -DestinationPath "backend.zip"

# 压缩完整项目
Compress-Archive -Path "frontend", "backend", "README.md" -DestinationPath "qxc_full.zip"
```

### 解压
```powershell
# 解压前端
Expand-Archive -Path "frontend.zip" -DestinationPath "extracted_frontend"

# 解压后端
Expand-Archive -Path "backend.zip" -DestinationPath "extracted_backend"
```

- Git

## 注意事项

1. **依赖管理**: 前端和后端使用不同的包管理器，需要分别安装依赖
2. **环境配置**: 确保环境变量和配置文件正确设置
3. **数据库**: 后端包含 SQLite 数据库文件，注意备份
4. **文件上传**: 上传目录包含用户文件，注意权限设置
5. **构建输出**: 构建文件较大，建议在 .gitignore 中排除
